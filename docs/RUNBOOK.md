## 🔔 2. متابعة الطلبات الجديدة

كل طلب جديد يصدر صوت تنبيه ويظهر badge أحمر على tab **الطلبات** في الـ sidebar.

**التدفق الطبيعي للطلب:**

1. **NEW (طلب جديد)** ← العميل أنشأ الطلب الآن
2. **UNDER_REVIEW (قيد المراجعة)** ← اضغط زر "بدء المراجعة" بعد قراءة التفاصيل
3. **PRICED (تم التسعير)** ← أدخل السعر النهائي وأي ملاحظات
4. **AWAITING_CUSTOMER_APPROVAL** ← انتظر حتى يوافق العميل من التطبيق
5. **ACCEPTED** ← اختر السائق المناسب وكلّفه
6. **DRIVER_ASSIGNED** ← السائق يستلم الإشعار في تطبيقه
7. **PICKED_UP / IN_ROUTE / DELIVERED / COMPLETED** ← يتحدثون تلقائياً من جهاز السائق

> 💡 يمكن استخدام **زر القائمة السريعة** بجوار كل طلب في القائمة الرئيسية لتغيير الحالة بضغطة واحدة بدون فتح صفحة التفاصيل.

## 🔐 2.1 ربط EasyKash (بوابة الدفع)

EasyKash هي بوابة الدفع الإلكتروني الوحيدة في تميم — تدعم فودافون كاش، InstaPay، Visa، MasterCard، وMeeza من نفس الصفحة.

**خطوات الربط لمرة واحدة** (تتم من إدارة العميل):

1. ادخل على https://back.easykash.net وأنشئ حساب التاجر.
2. اتبع الـ documentation: https://easykash.gitbook.io/easykash-apis-documentation
3. من **Integration Settings** انسخ الـ **API Key**.
4. في خانة **Callback URL** أدخل:
   `https://api.deliverytamem.com/api/v1/payments/webhook/easykash`
   (الـ URL لازم يكون GET-able من EasyKash للاختبار في Postman أولاً).
5. اختر طرق الدفع المطلوبة (Vodafone Cash، InstaPay، Visa، MasterCard، Meeza) ثم Save.
6. بعد الحفظ سيظهر **HMAC Secret Key** — انسخه.
7. في الداشبورد، اذهب إلى **بوابة الدفع** والصق:
   - API Key
   - HMAC Secret
   - فعّل طرق الدفع المرغوبة
8. اضغط **حفظ** ثم **اختبار الاتصال** للتأكد.

> 🔄 إذا غُيّر الـ API key لأي سبب، يكفي تحديثه من الداشبورد دون إعادة تشغيل السيرفر — التغيير يسري فوراً.

## 💳 3. تأكيد المدفوعات اليدوية (لو العميل دفع خارج EasyKash)

1. اذهب إلى **المدفوعات**
2. يظهر طلب الدفع مع صورة الإيصال
3. تحقق من الرقم المرجعي مقابل حساب فودافون كاش الخاص بتميم
4. اضغط ✅ "تأكيد" → تتحول الحالة إلى PAID → يصل إشعار للعميل
5. لو الإيصال خاطئ: اضغط ❌ "رفض" → اكتب السبب → يصل تنبيه للعميل

## ⚠️ 4. حل التنبيهات (Alerts Center)

- **PENDING_ORDER** = طلب مسعّر لم يوافق العميل عليه خلال 60 دقيقة → اتصل بالعميل
- **DRIVER_NOT_RESPONDING** = سائق لم يحدّث موقعه أكثر من 25 دقيقة → اتصل بالسائق
- **CASH_LIMIT_EXCEEDED** = السائق يحمل كاش > 1000 ج.م → اطلب تسليم الكاش للمكتب
- **PAYMENT_PENDING** = دفعة معلقة في Paymob أكثر من ساعة → راجع dashboard Paymob

## 📱 5. إعادة ربط واتساب (WppConnect QR re-scan)

عند إعادة تشغيل السيرفر قد ينقطع اتصال WppConnect ويحتاج مسح QR من جديد:

1. اذهب إلى **واتساب** من القائمة الجانبية
2. ستجد QR code معروض على الشاشة
3. افتح واتساب على هاتف الأدمن → الإعدادات → الأجهزة المرتبطة → ربط جهاز
4. وجّه الكاميرا للـ QR على الشاشة
5. تتغير الحالة إلى ✅ **متصل** خلال ثوان
6. إذا لم يظهر QR، اضغط زر **"بدء جلسة جديدة"** وأعد المحاولة

> 🔄 إذا تعطّل WppConnect باستمرار، يمكن تفعيل WhatsApp Cloud API كمزود أساسي بإضافة `WHATSAPP_PRIMARY=cloud` في ملف `.env` ثم إعادة تشغيل السيرفر.

## ➕ 6. إضافة خدمة جديدة

اطلع على `docs/SERVICE-BUILDER.md` للخطوات التفصيلية + screenshots.

ملخص:

1. اذهب إلى **الخدمات** → **+ خدمة جديدة**
2. املأ: الاسم، الصورة، الفئة، طريقة التسعير، السعر الأساسي
3. أضف الحقول التي يحتاجها العميل (نص، صورة، موقع، رقم، …)
4. حدد أيها مطلوب
5. اضغط **حفظ** → تظهر فوراً في تطبيق العميل بدون أي تحديث برمجي

## 🛠️ 7. إعادة تشغيل السيرفر

إذا توقف الـ API (تفاجأ بأن الموقع لا يستجيب):

```bash
ssh tamem@<server-ip>
pm2 status                # تأكد من حالة tamem-api
pm2 restart tamem-api     # إعادة تشغيل
pm2 logs tamem-api --lines 100   # متابعة السجلات
```

## 💾 8. النسخ الاحتياطي + الاسترجاع

- النسخ الاحتياطي يعمل تلقائياً كل ليلة 3:15 صباحاً (`/etc/cron.d/tamem-backup`).
- النسخ مشفّرة في `/var/backups/tamem/db-YYYY-MM-DD.sql.gz.enc` (تحتفظ بآخر 14 يوم).
- المرفقات (uploads) يتم نسخها أسبوعياً كل أحد 3:30 صباحاً.

### استرجاع DB

```bash
# 1. فك التشفير
openssl enc -d -aes-256-cbc -pbkdf2 -pass file:/home/tamem/.backup-key \
  -in /var/backups/tamem/db-2026-06-03.sql.gz.enc \
  -out /tmp/db-restore.sql.gz

# 2. استيراد
gunzip -c /tmp/db-restore.sql.gz | mysql -uroot -p tamem

# 3. تشغيل الـ migrations الجديدة (إن وجدت)
cd /var/www/tamem && pnpm --filter @tamem/backend prisma migrate deploy
```

## 🚨 10. حالات الطوارئ

| الحالة               | الإجراء الفوري                                                                     |
| -------------------- | ---------------------------------------------------------------------------------- |
| الموقع لا يفتح أصلاً | تحقق من UptimeRobot في WhatsApp، ثم `pm2 status` على VPS                           |
| الواتساب لا يرسل     | افتح dashboard `/whatsapp` → re-scan QR                                            |
| الدفع لا يكتمل       | تحقق من Paymob dashboard. لو الموقع نفسه يعمل لكن webhook لا يصل → راجع nginx logs |
| طلب عالق في NEW      | اضغط Force Refresh + تأكد من Socket.IO يعمل (badge "متصل" أعلى الـ sidebar)        |
| DB تالفة             | استرجع آخر نسخة احتياطية (الخطوة 8) ثم اتصل بالدعم                                 |
| فقد بيانات الأدمن    | استخدم 1Password (admin-credentials.tamem)                                         |

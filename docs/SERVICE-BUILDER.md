# بناء خدمة جديدة من الداشبورد

> دليل خطوة-بخطوة لإدارة تميم لإضافة خدمة جديدة تظهر فوراً في تطبيق العميل بدون أي تدخل برمجي.

النظام بُنى حول فكرة **الخدمات الديناميكية**: الأدمن يحدد اسم الخدمة + صورتها + التسعير + الحقول التي يملأها العميل، والتطبيق يرسم النموذج تلقائياً من تعريف الخدمة.

---

## 1. الدخول

من القائمة الجانبية → **الخدمات** → **+ خدمة جديدة**.

## 2. البيانات الأساسية

| الحقل                | المعنى                     | مثال                |
| -------------------- | -------------------------- | ------------------- |
| **الاسم بالعربي**    | يظهر للعميل في الواجهة     | "توصيل مستندات"     |
| **الاسم بالإنجليزي** | للسجلات والتوثيق           | "Document Delivery" |
| **المفتاح (key)**    | معرّف فريد بدون مسافات     | "doc-delivery"      |
| **الفئة**            | دليفري / شحن / تاجر        | DELIVERY            |
| **الصورة**           | تظهر في الهوم على الموبايل | اختر من جهازك       |

## 3. طريقة التسعير

اختر واحدة:

- **FIXED** — سعر ثابت (مثلاً 30 ج.م لكل توصيلة)
- **DISTANCE** — حسب المسافة (Base + Per km)
- **WEIGHT** — حسب الوزن (Base + Per kg) — للشحن
- **DISTANCE_WEIGHT** — المسافة + الوزن — للشحن بين المدن
- **QUOTE** — الإدارة تسعّر يدوياً (للخدمات المعقدة كحجز دكتور أو طلبات مخصصة)

> 💡 يمكن إضافة قواعد تسعير تفصيلية لكل خدمة من صفحة **قواعد التسعير** (حسب المحافظة، عمولة المساء، رسوم العاجل، الحد الأقصى، إلخ).

## 4. خيارات سلوك الخدمة

- ☑️ **يتطلب موقع الاستلام؟** للخدمات التي تستلم من مكان (مطعم، صيدلية)
- ☑️ **يتطلب موقع التسليم؟** افتراضياً ✓
- ☑️ **يدعم استلام من نقاط متعددة؟** للتجار B2B
- ☑️ **يدعم تسليم لنقاط متعددة؟** للشحن الجماعي
- ☑️ **رفع صور إجباري؟**
- ☑️ **ملاحظات نصية مسموحة؟**

## 5. الحقول المخصصة (Service Fields)

اضغط **+ حقل** لإضافة حقل جديد. لكل حقل اختر:

| نوع الحقل   | متى تستخدمه     | مثال                             |
| ----------- | --------------- | -------------------------------- |
| TEXT        | نص قصير         | "اسم الدواء"                     |
| TEXTAREA    | نص طويل         | "ملاحظات الطلب"                  |
| NUMBER      | رقم             | "الكمية"                         |
| PHONE       | رقم هاتف        | "رقم المستلم"                    |
| SELECT      | اختيار من قائمة | "حجم الطرد: صغير / متوسط / كبير" |
| MULTISELECT | اختيارات متعددة | "ملحقات الطلب"                   |
| BOOLEAN     | نعم / لا        | "هل يحتاج تبريد؟"                |
| IMAGE       | صورة            | "صورة الروشتة"                   |
| LOCATION    | موقع جغرافي     | "موقع التسليم"                   |
| DATE        | تاريخ           | "تاريخ الميلاد"                  |
| TIME        | وقت             | "وقت التسليم المفضل"             |

لكل حقل حدد:

- **الاسم بالعربي** (يظهر للعميل)
- **المفتاح** (يُستخدم برمجياً)
- **مطلوب؟** (تجبر العميل على ملئه)
- **ترتيب الظهور** (سهم ↑↓ لإعادة الترتيب)
- **خيارات إضافية** حسب نوع الحقل (مثلاً عناصر القائمة لـ SELECT)

## 6. المعاينة الحية

في الأسفل تظهر معاينة فورية للنموذج كما سيراه العميل في الموبايل — أي تغيير على الحقول يُحدّث المعاينة فوراً.

## 7. الحفظ

اضغط **حفظ** أعلى الصفحة. تُحفظ الخدمة في قاعدة البيانات وتظهر في تطبيق العميل خلال 30 ثانية (وقت التخزين المؤقت).

## 8. التعديل لاحقاً

من **الخدمات** → اضغط على اسم الخدمة → **تعديل**. كل التغييرات تتم بـ Auto-save (debounce 1s) — لا تحتاج لزر حفظ منفصل.

## 9. الإيقاف المؤقت

من نفس الصفحة، اضغط **إيقاف** بدل الحذف. الخدمات الموقوفة لا تظهر للعميل لكن الطلبات السابقة تبقى محفوظة.

## أمثلة عملية

### مثال 1: خدمة "توصيل وجبة" (FIXED)

- **التسعير:** FIXED، Base price = 25 ج.م
- **الحقول:**
  - LOCATION: "أين تستلم الوجبة؟" (موقع الاستلام)
  - LOCATION: "أين توصل لك؟" (موقع التسليم) — مطلوب
  - TEXTAREA: "ملاحظات للسائق"
- **الخيارات:** يتطلب موقع الاستلام + يتطلب موقع التسليم

### مثال 2: خدمة "شحن طرد بين المحافظات" (DISTANCE_WEIGHT)

- **التسعير:** DISTANCE_WEIGHT، Base = 30، Per km = 1.5، Per kg = 2
- **الحقول:**
  - LOCATION: "من" (مطلوب)
  - LOCATION: "إلى" (مطلوب)
  - NUMBER: "الوزن بالكيلو" (مطلوب)
  - SELECT: "الحجم" — صغير / متوسط / كبير
  - BOOLEAN: "قابل للكسر؟"
  - SELECT: "سرعة التوصيل" — عادي / عاجل
  - IMAGE: "صورة الطرد"

### مثال 3: خدمة "حجز دكتور" (QUOTE)

- **التسعير:** QUOTE — الإدارة تتواصل وتسعّر يدوياً
- **الحقول:**
  - TEXT: "اسم الدكتور أو التخصص" (مطلوب)
  - DATE: "التاريخ المفضل"
  - TIME: "الوقت المفضل"
  - SELECT: "العيادة المفضلة"
  - PHONE: "رقم تواصل" (مطلوب)

## النصائح

- لا تستخدم مفاتيح (keys) فيها رموز أو مسافات — استخدم kebab-case (`pickup-location`).
- لا تحذف حقلاً عليه طلبات قديمة — اضغط "إخفاء" بدلاً منه. حتى لو لم يظهر للعميل الجديد، البيانات القديمة تبقى متاحة في التقارير.
- ابدأ بحقول قليلة (3-4) ثم أضف لاحقاً — كل حقل إضافي يقلل من معدل إكمال الطلب.
- استخدم helpText بالعربي لتوضيح الحقول المعقدة.
